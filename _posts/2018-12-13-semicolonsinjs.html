The semicolon in JavaScript is used to separate statements, but it can be omitted if the statement is followed by a line break (or thereâ€™s only one statement in a {block})

They are optional in Javascript

Javascript will automatically insert semi colons for you using ASI(automatic semi colon insertion) if you avoid them.

Reasons for using semi colons
  • They are required when two statements are on same line to separate the lines.
  • If your team is using it, then you can use it for uniformity.

 

Some invalid reasons for using semi colon
  • Thinking that it will protect you from ASI errors

Adding semi colons will not help you in the following error. The ASI will insert a semi colon after the return statement.

{{site.js}} //Example of one error caused by automatic semi colon insertion function returnA () { return 4 } //4 function returnB () { return 4 } //undefined //Second function is interpreted like this function returnC () { return; 4; } console.log(returnA(), returnB(), returnC(),) {{site.cbe}}
Errors caused by not using semi colons

There are a handful of situations when not using semicolons will land you in trouble. I have listed two such cases here.

1.When Looping through an array

{{site.cbs}}//This will give an error [1,2,3].forEach {{site.cbe}}

It can be solved by inserting a semi colon at the beginning

{{site.cbs}}//Add Semi colon at the beginning ;[1,2,3].forEach {{site.cbe}}

or

Assign array to a variable and use forEach {{site.cbs}} const arr=[1,2,3] arr.forEach {{site.cbe}}

2.When using an IIFE(Immediately Invoked Function Expression)

{{site.cbs}}(function(){ //function body }()); //This will produce error. // Can be resolved using a semi colon ;(function(){ //function body }()); {{site.cbe}}
Situations to avoid semi colons

After a closing curly bracket except for an assignment statement.

{{site.cbs}}// NO semicolons after }: if (...) {...} else {...} for (...) {...} while (...) {...} // BUT: do {...} while (...); // function statement: function (arg) { /*do this*/ } // NO semicolon after } {{site.cbe}}

After the round bracket of an if, for, while or switch statement

{{site.cbs}}if (0 === 1); { alert("hi") } // equivalent to: if (0 === 1) /*do nothing*/ ; alert ("hi"); {{site.cbe}}
The less confusing path and what I have chosen regarding semi colons
  • Assign your arrays to variable and loop through them.
  • Use a semi colon before IIFE.
  • Don't use semi colons other than that.
